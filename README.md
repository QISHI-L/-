# 咸鱼AI｜AI 原生助手

> 面向闲鱼商家的智能运营后台，重点解决 **自动回复** 和 **自动发货** 两个高频问题，让店铺从“人盯人”转向“系统自动跑”。

## 这是什么

咸鱼AI 是一个为闲鱼卖家打造的运营管理平台。它把账号接入、商品管理、订单处理、自动回复、自动评价等能力集中到一个后台里，帮助商家减少重复操作，提高响应速度，稳定处理日常交易流程。

如果你每天都在反复做这些事：
- 回同样的问题
- 盯着订单等发货
- 手动处理商品配置
- 来回切换多个店铺账号

这个系统的价值就很直接：**把重复工作交给系统，把时间留给成交和运营。**

## 适合谁使用

- 闲鱼个人卖家
- 做数字商品、资料商品、会员商品的商家
- 代运营团队
- 需要多账号协同管理的运营人员
- 想提升客服效率和发货效率的店铺

## 核心卖点

### 1. 自动回复，减少重复客服工作

面对“在吗”“什么时候发货”“怎么使用”“还有没有货”这类高频问题，人工一条条回复既慢又累。咸鱼AI 支持通过 **关键词规则** 和 **自动回复策略** 来处理常见咨询，让买家更快收到回应。

你可以把它理解为一个面向闲鱼场景的客服助手：
- 对常见问题自动响应
- 按关键词匹配不同回复内容
- 支持下单后自动发送说明信息
- 支持每句话自动回复
- 可结合 AI 上下文设置统一回复口径

这意味着：
- 响应更快
- 咨询不容易漏
- 店铺回复更统一
- 人工客服压力更小

### 2. 自动发货，缩短成交后的处理时间

对于数字商品、卡券、资料包、会员服务等商品，成交后最怕的就是还要手动盯单、复制内容、逐个发送。咸鱼AI 支持在商品维度配置 **自动发货内容**，让系统在合适的流程里自动完成交付。

可见能力包括：
- 商品级设置自动发货内容
- 配置自动发货开关
- 联动商品与订单处理流程
- 配合自动确认、自动求花等功能形成完整自动化链路

对卖家来说，最大的变化不是“多了一个功能”，而是：
- 买家等待时间更短
- 你不用一直守着后台
- 高峰时段也能稳定处理订单
- 更适合批量商品和重复交付场景

## 功能模块一览

### 闲鱼帐号管理

![闲鱼帐号管理](docs/screenshots/account-management.png)

这是系统的基础入口。账号接入完成后，后续的商品、订单、自动化规则才能正常运转。

**可见功能：**
- 支持手动输入 CookieStr
- 支持扫码添加 CookieStr
- 支持账号密码登录
- 查看 Cookie 状态、AccessToken 状态、实例状态
- 支持启动实例、停止实例、删除 Cookie
- 支持设置账密自动刷新 Cookie
- 支持手动刷新 AccessToken
- 支持拉取店铺信息并展示店铺基础数据

**客户价值：**
- 统一管理多个店铺账号
- 快速判断账号是否可用
- 为自动发货和自动回复提供运行基础

---

### 商品管理

![商品管理](docs/screenshots/product-management.png)

商品管理是整个平台最核心的业务页面之一。这里不仅能看商品，更重要的是能把自动化能力直接配置到商品上。

**可见功能：**
- 按店铺切换查看商品
- 按商品名称搜索
- 新增商品
- 拉取所选账号商品
- 刷新商品列表
- 针对商品开启/关闭自动发货
- 开启/关闭自动确认
- 开启/关闭自动求花
- 开启/关闭自动 AI 回复
- 开启/关闭自动擦亮
- 设置自动发货内容
- 设置 AI 上下文

**客户价值：**
- 一个页面完成商品与自动化配置
- 自动发货内容可以跟商品绑定
- 自动回复和 AI 上下文可以按商品精细设置
- 适合数字商品、资料商品、虚拟交付类业务

---

### 关键词回复规则

![关键词回复规则](docs/screenshots/keyword-rules.png)

当买家咨询内容高度重复时，关键词规则是最直接的提效方式。你可以为不同商品、不同规则名、不同关键词设置对应回复文本。

**可见功能：**
- 新增关键词规则
- 按商品 ID 筛选
- 按规则名搜索
- 按关键词搜索
- 按启用状态筛选
- 配置关键词与自动回复文本

**客户价值：**
- 常见问题自动处理
- 首次响应更快
- 统一店铺话术，减少回复差异
- 是自动回复体系的重要基础模块

**客户价值：**

- 减少客服重复发送固定内容
- 提升买家收到信息的及时性
- 进一步强化自动回复能力

![自动评价管理](docs/screenshots/auto-review.png)

- 设置默认评价内容
- 设置求好评内容
- 查看启用状态
- 支持编辑与删除配置
- 支持新增配置

**客户价值：**
- 统一评价文案
- 减少人工补评价操作
- 完善成交后的自动化闭环

### 订单管理

![订单管理](docs/screenshots/order-management.png)

订单管理负责承接实际交易结果。商家可以集中查看订单状态，并处理相关动作。

**可见功能：**
- 按店铺查看订单
- 按订单状态筛选
- 刷新订单
- 展示下单时间、订单号、商品标题、金额、订单状态、卖家信息
- 提供订单相关操作入口

**客户价值：**
- 快速查看订单进展
- 方便核对交易结果
- 与自动发货场景形成联动
- 降低订单处理混乱度

## 一套更顺的运营流程

对于客户来说，这个平台不是“功能堆叠”，而是一条更顺的运营链路：

1. 先在 **闲鱼帐号** 中接入店铺账号
2. 拉取商品与店铺数据
3. 在 **商品管理** 中配置自动发货、自动确认、AI 回复等能力
4. 在 **关键词回复规则** 中设置常见咨询的自动回复话术
5. 在 **自动回复管理** 中补充店铺级回复策略
6. 在 **订单管理** 中查看成交结果和订单状态
7. 在 **自动评价管理** 中补全售后互动流程

## 为什么客户会关心这个系统

因为它解决的不是抽象问题，而是每天都在发生的真实工作：

- 买家来了，能不能第一时间回复
- 订单成交后，能不能尽快发货
- 商品多了，能不能统一管理
- 店铺多了，能不能少切页面
- 业务跑起来后，能不能减少人工盯盘

如果你的店铺已经开始有稳定咨询和订单，这类系统的意义通常很明确：
**减少重复动作，提升处理速度，让运营更轻。**
